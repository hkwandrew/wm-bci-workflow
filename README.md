# WM BCI Workflow

`WM BCI Workflow` is a WordPress plugin that turns one Gravity Forms submission flow into a managed BCI publishing workflow:

1. A community opportunity is submitted through Gravity Forms.
2. The entry is marked `Pending` by default, or `Approved` automatically for allowlisted WordPress users.
3. Reviewers get a formatted notification email with secure approve/reject links.
4. Only approved entries appear in the configured GravityCalendar feed.
5. Approved entries can be exported as CSV and, when configured, synced to a Google Sheet used for the newsletter workflow.

The plugin is intentionally centered on one form and one approval pipeline. Most behavior is controlled from `Settings > BCI Workflow`.

## Features

- Seeds new Gravity Forms entries into an approval workflow.
- Auto-approves future submissions from selected logged-in WordPress users.
- Suppresses the review notification when an entry is auto-approved.
- Copies the `Grant / RFP` deadline into the calendar start-date field when needed.
- Rewrites the configured Gravity Forms notification into a reviewer-friendly HTML email.
- Generates secure review links for approving or rejecting an entry without opening the WP admin first.
- Filters GravityCalendar events so only approved entries appear on the public BCI resources page.
- Adds custom calendar tooltip markup, layout options, and per-opportunity-type color controls.
- Exports approved opportunities as CSV in the same column order used by the Google Sheet workflow.
- Syncs approved opportunities to a Google Apps Script endpoint backed by Google Sheets.
- Adds a dashboard widget with sync status counts, retry controls, CSV import, and CSV export.
- Supports a one-time CSV import from an existing Google Sheet so existing rows can be marked as already synced.

## Requirements

- WordPress
- PHP 8.2 or newer for the installed plugin runtime
- Gravity Forms
- A Gravity Forms form that stores the fields mapped in this plugin's settings

Optional integrations:

- GravityKit / GravityView Calendar feed support for public calendar output
- Google Apps Script plus Google Sheets for automatic sync

If Gravity Forms is not active, the plugin does not boot and shows an admin error notice.

## High-Level Workflow

### 1. Submission intake

When a new entry is saved for the configured Gravity Form:

- `src/Approval/EntrySeeder.php` sets the approval field to `Pending` if it is blank.
- If the submitter's `created_by` user ID is on the auto-approved list, the entry is marked `Approved` immediately.
- For `Grant / RFP` entries, the grant deadline can be copied into the calendar start date field if the start date is blank.

### 2. Review email

`src/Notification/ApprovalEmail.php` customizes the configured Gravity Forms notification by:

- optionally overriding the recipient list from plugin settings
- building a structured HTML summary of the submission
- adding secure `Approve submission` and `Reject submission` links
- linking directly to the Gravity Forms entry in wp-admin

The review links are generated by `src/Approval/ReviewUrl.php` and expire after 7 days.

### 3. Approval handling

`src/Approval/ReviewHandler.php` receives approve/reject requests through `admin-post.php` and:

- validates the signed URL
- updates the configured approval field
- stores an approval timestamp when an entry becomes approved
- records a Gravity Forms note
- triggers Google Sheet sync when an entry moves to `Approved`

### 4. Calendar publishing

The calendar integration is split across several classes:

- `src/Calendar/EventFilter.php`: limits the configured feed to approved entries only
- `src/Calendar/EventCustomizer.php`: replaces the event description with tooltip markup and applies configured event colors
- `src/Calendar/TooltipOptions.php`: adjusts FullCalendar and tooltip options for the BCI feed
- `src/Calendar/Styles.php`: loads front-end tooltip styles on the configured calendar page

### 5. Export and sync

Approved entries are mapped by `src/Export/RowMapper.php`, which is shared by:

- `src/Export/CsvExporter.php`
- `src/GoogleSync/SyncManager.php`
- `src/GoogleSync/CsvImporter.php`

That shared row mapping is important because the manual CSV export, the Google Sheet sync payload, and the CSV import all depend on the same column contract.

## Installation

1. Install the plugin in `wp-content/plugins/wm-bci-workflow`.
2. Ensure Gravity Forms is active.
3. Activate `WM BCI Workflow`.
4. Open `Settings > BCI Workflow`.
5. Confirm the configured form ID, approval field ID, field map, calendar settings, and optional Google sync settings.

On activation, the plugin creates the `wm_bci_workflow` option if it does not already exist.

## Settings

All settings are stored in the `wm_bci_workflow` option and accessed through `src/Config.php`.

### Form Configuration

These fields appear in `Settings > BCI Workflow`:

- `Form ID`
  - Defaults to `4`.
  - The plugin only acts on this Gravity Forms form.
- `Approval Field ID`
  - Defaults to `22`.
  - Stores `Pending`, `Approved`, or `Rejected`.
- `Notification Name`
  - Defaults to `Admin Notification`.
  - Only this Gravity Forms notification is rewritten for review email behavior.
- `Approval Notification Recipients`
  - Optional comma-separated or newline-separated email list.
  - When provided, this overrides the Gravity Forms `Send To` destination for the review email.
- `Auto-Approved Submitters`
  - Multi-select of WordPress users.
  - Future entries created by those logged-in users are approved automatically.
  - This only works when Gravity Forms records `created_by` on the entry.
- `Calendar Page Slug`
  - Defaults to `bci-resources`.
  - Used to decide where front-end tooltip styles should load.
- `Calendar Feed Name`
  - Defaults to `BCI Community Opportunity Submission`.
  - Used to identify the GravityCalendar feed that should be filtered to approved entries.

### Field Mapping

The plugin maps semantic field keys to Gravity Forms field IDs. Current defaults are:

| Key | Default field ID |
| --- | --- |
| `opportunity_type` | `1` |
| `submitter_name` | `3` |
| `title` | `4` |
| `organization` | `5` |
| `start_date` | `6` |
| `grant_deadline` | `9` |
| `end_date` | `10` |
| `start_time` | `12` |
| `cost` | `14` |
| `address` | `15` |
| `description` | `17` |
| `info_url` | `18` |
| `file_upload` | `19` |
| `end_time` | `21` |
| `approval_status` | `22` |

These values are read through `Config::field()` and normalized through `src/Entry/FieldAccessor.php`.

### Calendar Event Colors

The settings page can assign a fixed event color to each opportunity type choice found on the configured `opportunity_type` field.

Notes:

- Only the fixed palette defined in `Config::calendar_event_palette()` is accepted.
- If no color is selected for a type, GravityCalendar's default event color is used.
- If the opportunity-type field supports an `Other` choice, the settings UI can store a color for `Other` as well.

### Google Sheets Sync

The settings page accepts:

- `Sync Endpoint URL`
- `Shared Secret`

The secret field is write-only in the UI. Leaving it blank preserves the existing saved secret.

The plugin can also fall back to these constants in `wp-config.php`:

```php
define( 'WATERS_MEET_BCI_GOOGLE_SYNC_URL', 'https://script.google.com/...' );
define( 'WATERS_MEET_BCI_GOOGLE_SYNC_SECRET', 'your-shared-secret' );
```

If both a saved option and a constant exist, the saved option is used first.

## Auto-Approved Submitters

Auto-approval is handled in `src/Approval/EntrySeeder.php`.

Behavior:

- The selected WordPress user IDs are stored as `auto_approved_user_ids`.
- When a new entry is created by one of those users, the approval field is set to `Approved`.
- The plugin stores the approval timestamp immediately.
- A Gravity Forms note is added explaining that the entry was auto-approved.
- The configured review notification is suppressed for that entry.
- If Google sync is configured, the entry is synced immediately after auto-approval.

This is designed for trusted internal submitters who should bypass the manual review step.

## Review Links

Approval links are generated by `src/Approval/ReviewUrl.php`.

Properties of the link flow:

- Requests go through `admin-post.php?action=wm_bci_review`.
- The link contains the entry ID, target status, expiration timestamp, and an HMAC signature.
- Signatures are derived from the form ID, entry ID, status, expiration timestamp, and `wp_salt( 'auth' )`.
- Links expire after `WEEK_IN_SECONDS` (7 days).

If a link is missing parameters, expired, or has an invalid signature, the reviewer sees a plain success/error page instead of a fatal error.

## Calendar Behavior

The plugin assumes one BCI feed should be public and approval-gated.

### Approved-entry filtering

`src/Calendar/EventFilter.php` appends an `Approved` condition to the matching feed if that condition is not already present.

The feed match requires both:

- the configured Gravity Form ID
- the configured calendar feed name

### Tooltip rendering

`src/Calendar/EventCustomizer.php` builds custom tooltip markup that can include:

- opportunity type
- date or deadline
- time
- organization
- location
- a shortened description
- a link to the full detail URL

It also applies per-type event colors and chooses light or dark text based on color luminance.

### Front-end presentation

`src/Calendar/TooltipOptions.php` and `src/Calendar/Styles.php` set calendar display behavior and load front-end styles on the configured calendar page only.

## Dashboard Widget

The dashboard widget is registered by `src/Dashboard/Widget.php` and shown to users who can export Gravity Forms entries.

It provides:

- a summary of approved/synced, approved/awaiting sync, failed sync, and pending review counts
- a retry button for unsynced or failed approved entries
- a one-time CSV import form for seeding previously synced Google Sheet rows
- a CSV export button
- a quick link to the form's entry list

Related feedback messages are rendered by `src/Dashboard/AdminNotice.php`.

## CSV Export Contract

`src/Export/RowMapper.php` defines the canonical export columns and row shape.

The current column set is:

1. `Timestamp`
2. `What kind of opportunity is this?`
3. `Your name:`
4. `What is the title of your community opportunity?`
5. `What is the name of your organization?`
6. `When is your opportunity happening? We need this to have this with at least a week's notice. The newsletter goes out on Thursdays, so anything happening before Thursday of the current week should not be included.`
7. `If your opportunity has a date range, what is the end date?`
8. `For events and learning opportunities, what time of day is it happening?`
9. `For opportunities with a physical location, what is the address?`
10. `Is there any cost?`
11. `Provide a short description of this opportunity`
12. `Provide a link for additional information:`
13. `Please upload any relevant files here:`
14. `Has this been in a newsletter?`
15. `Additional Info , Instructions, and Commentary`

The CSV exporter:

- includes a UTF-8 BOM
- exports approved entries only
- sorts rows by primary date and then by title
- names the download `bci-approved-opportunities-YYYY-MM-DD.csv`

## Google Sheet Sync

Automatic sync is owned by `src/GoogleSync/SyncManager.php`.

### When sync runs

Sync is attempted when:

- an entry is auto-approved on submission
- a reviewer approves an entry via a review link
- an admin updates an entry to `Approved`
- a user clicks `Retry Google Sheet Sync` in the dashboard widget

The update hooks live in `src/GoogleSync/SyncTrigger.php`.

### What sync sends

The outgoing payload includes:

- event name
- entry ID
- approval timestamp
- header row
- row data
- wp-admin URL for the entry
- wp-admin URL for the full entry list

The request:

- is sent as JSON
- uses `POST`
- signs the JSON body with HMAC SHA-256 using the shared secret
- includes the signature both as a query arg and as `X-Waters-Meet-Signature`

### Success and failure rules

Sync only succeeds when the endpoint returns:

- an HTTP 2xx status
- JSON with `ok: true`
- a recognized `disposition` of `appended` or `duplicate`

On success the plugin stores sync metadata and adds a Gravity Forms note.

On failure the plugin stores failure metadata and adds a Gravity Forms note with the error message.

### Google Apps Script setup

The plugin ships two support files for the Google Sheet integration:

- `support/google-sheet-sync.gs`
- `support/google-sheet-sync-setup.md`

Use the setup guide for the one-time Apps Script and spreadsheet configuration.

## CSV Import for Existing Sheet Data

`src/GoogleSync/CsvImportHandler.php` and `src/GoogleSync/CsvImporter.php` support a one-time import of rows that already exist in the connected Google Sheet.

This is useful when the sheet already has historical rows and you want WordPress to treat them as already synced.

Import behavior:

- accepts only uploaded `.csv` files
- limits uploads to 2 MB
- requires the header row to exactly match the export contract
- skips blank rows
- deduplicates imported rows by hashing the ordered row values
- creates approved Gravity Forms entries from the CSV rows
- marks those imported entries as already synced
- stores import and sync metadata so retries do not re-send them immediately

Each imported entry receives a Gravity Forms note:

- `Imported from a Google Sheet CSV upload and marked as already synced.`


Current test coverage includes:

The plugin keeps most behavior in small classes registered from `src/Plugin.php` rather than one large loader file.

- admin-post handlers for explicit export, retry, import, and review actions

## Troubleshooting

### The plugin shows a Gravity Forms dependency notice

Gravity Forms is required at boot. Activate Gravity Forms first, then reload wp-admin.

### Entries are not appearing on the public calendar

Check:

- the plugin `Form ID`
- the `Approval Field ID`
- the entry's actual approval value
- the `Calendar Feed Name`
- whether the public page matches the configured `Calendar Page Slug`

### Auto-approval is not happening for a trusted user

Check:

- the user is selected in `Auto-Approved Submitters`
- the form submission happened while that user was logged in
- Gravity Forms saved the entry with a valid `created_by` value

### Review emails are going to the wrong place

Check:

- the configured `Notification Name`
- whether `Approval Notification Recipients` is blank or overriding the Gravity Forms notification recipient

### Google Sheet sync is failing

Check:

- the sync URL
- the shared secret
- that the Apps Script deployment is still active
- that the Apps Script endpoint returns JSON with `ok: true`
- the Gravity Forms entry notes for messages added by `WM BCI Google Sync`

### CSV import fails immediately

Check:

- the uploaded file is a real `.csv`
- the file is 2 MB or smaller
- the header row exactly matches the export contract
